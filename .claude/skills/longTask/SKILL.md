# Long Task - 장시간 자율 실행 모드

## 1단계: 사전 질문

작업을 시작하기 전에 다음을 반드시 파악한다. 부족한 정보는 사용자에게 질문한다.

### 필수 파악 항목
- **목표**: 최종 달성 조건이 무엇인가? (예: "8개 빌드 케이스 전부 SUCCESS")
- **제약**: 하면 안 되는 것이 있는가? (예: "엔진 소스 수정 금지")
- **우선순위**: 상충하는 요구사항이 있을 때 무엇을 우선하는가?
- **실패 시 행동**: 몇 회 재시도? 해결 불가 시 보고만? 대안 시도?

### 사전 질문 규칙
- 목표가 명확하면 불필요한 질문을 하지 않는다
- 사용자가 이미 말한 내용을 다시 묻지 않는다
- 질문은 한 번에 모아서 한다 (하나씩 묻지 않는다)

## 2단계: 자율 실행

사전 질문이 완료되면 목표 달성까지 자율 실행한다.

### 실행 규칙
- 중간에 사용자에게 **묻지 않는다** (아래 예외 제외)
- 에러 발생 시 **스스로 원인 분석 → 수정 → 재시도** (최소 3회)
- 정보가 부족하면 코드/로그/문서를 직접 탐색해서 파악한다
- 하나의 단계가 완료되면 다음 단계로 바로 진행한다
- 진행 상황은 간결하게 보고하되, 확인을 요청하지 않는다

### 사용자 요구사항과 충돌 시
- 1단계에서 파악한 제약/의도에 반하는 행동이 필요한 경우에만 중단하고 질문한다
- 사용자가 명시적으로 요청한 작업을 번복해야 하는 상황이면 반드시 사전 질문한다
  - BAD: 사용자가 "삭제해"라고 한 것을 빌드 실패 후 묻지 않고 복원
  - GOOD: "삭제하신 X를 복원해야 빌드가 성공합니다. 복원해도 될까요?"

### 중단이 허용되는 경우 (이 경우에만 사용자에게 질문)
- 1단계에서 파악한 제약과 충돌하는 행동이 필요할 때
- 되돌릴 수 없는 파괴적 작업이 필요할 때
- 3회 이상 재시도해도 해결 불가능할 때 (시도한 내용과 함께 보고)

## 3단계: 결과 보고

목표 달성 또는 실패 시 결과를 보고한다.

### 보고 내용
- 목표 달성 여부
- 수행한 작업 요약
- 변경된 파일/설정 목록
- 미해결 사항 (있는 경우)

## Fact 기반 원칙 (CLAUDE.md 동일 적용)

자율 실행 중에도 가설을 사실처럼 말하지 않는다:
- 원인 분석 시 검증 전에 단정하지 않는다
- 수정 후 실행 결과로 검증한 뒤에만 "해결됨"으로 판단한다
- 문서 작성 시 미확인 사항은 "미확인", "추정" 명시
