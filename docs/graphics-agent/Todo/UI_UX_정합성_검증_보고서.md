# UI/UX 정합성 검증 보고서

검증일: 2026-03-07
검증 대상: 7개 화면 기획서 vs 실제 코드/리소스

---

## 1. 리소스 누락 현황

### 1-1. 펫 아이콘 (심각도: 높음)

- **현재 상태**: 18종 전량 미제작
- **코드 동작**: `SpriteManager.GetPetIcon()` 메서드 존재하나 `PetScreen`에서 미사용. 대신 이름 첫 글자를 텍스트로 표시하는 방식 사용 중
- **문제점**: 펫 화면 기획서에서는 "PetIcon 표시"를 명시하나, 실제로는 색상 배경 + 이니셜 텍스트만 표시
- **개선 방향**: 펫 아이콘 18종 제작 후, `PetScreen`에서 `SpriteManager.GetPetIcon()` 호출로 전환
- **영향 범위**: 그래픽 제작 + 코드 변경 필요
- **참조**: `docs/graphics-agent/리소스_제작_요청서/펫_아이콘.md` (제작 요청서 작성 완료)

### 1-2. UI 공통 스프라이트 4종 누락 (심각도: 중간)

- **현재 상태**: `buttonSprite`, `panelSprite`, `frameSprite`, `circleSprite` 미존재
- **코드 동작**: `SpriteDatabase`에 필드 정의되어 있고, `SpriteManager.GetUISprite()`에서 참조하나 null 반환
- **문제점**: 9-slice 버튼/패널 배경이 없어 코드에서 직접 `Image.color`로 단색 사각형 생성
- **개선 방향**: 9-slice 스프라이트 4종 제작 (둥근 모서리, 그림자, 테두리 포함)
- **영향 범위**: 그래픽만 (파일 추가 시 자동 적용)
- **참조**: `Assets/_Project/Resources/Icons/` 경로에 저장

### 1-3. player/wolf sprite.png 미존재 (심각도: 낮음)

- **현재 상태**: 프레임 애니메이션(idle/walk/attack)만 존재, 단일 스프라이트 없음
- **코드 동작**: `SpriteManager.LoadBattleSprite()`가 sprite.png 실패 시 `idle_0` 레거시 폴백 시도 후, 그것도 없으면 null 반환. 단, 프레임 애니메이션 로딩은 별도 경로로 정상 동작
- **문제점**: player와 wolf는 전투 시 프레임 애니메이션으로 표시되므로 sprite.png가 없어도 실질적 문제 없음. 다만 다른 시스템(아이콘 표시 등)에서 참조 시 null 가능
- **개선 방향**: 프레임 애니메이션의 idle 첫 프레임을 512x512로 리사이즈하여 sprite.png 생성 가능
- **영향 범위**: 그래픽만

## 3. 시각적 일관성 이슈

### 3-1. 펫 아이콘 vs 몬스터/장비 아이콘 불균형

- **문제**: 몬스터(53종), 장비(42종), 스킬(48종) 아이콘은 모두 전용 이미지가 있으나 펫(18종)만 텍스트 이니셜로 표시
- **영향**: 펫 화면만 다른 화면 대비 시각적 완성도가 현저히 낮음
- **권장 우선순위**: 최우선

### 3-2. UI 공통 스프라이트 부재

- **문제**: 버튼/패널이 모두 단색 사각형. 둥근 모서리, 그림자, 테두리 등 시각적 깊이감 없음
- **영향**: 전체 UI가 프로토타입 수준으로 보임
- **권장 우선순위**: 높음 (단, 코드가 이미 단색 사각형 기반으로 작동하므로 급하지는 않음)
