# 치트 검증 목록

## 작성일
2026-03-07

## 기획 의도
모바일 게임에서 메모리 에딧, 세이브 파일 조작, 패킷 변조, 속도핵 등 다양한 치트 벡터가 존재한다. 서버 권위 모델을 통해 클라이언트 값을 신뢰하지 않고, 서버에서 모든 중요 데이터를 관리하여 치트를 원천 차단한다.

## 목표
- 14개 도메인에 대한 치트 벡터 식별 및 서버 대응 방안 수립
- 48개 API 엔드포인트 설계
- 이상 감지 임계값 및 자동 처리 기준 정의

## 동작 방식

### 권위 모델 분류

| 기호 | 의미 |
|------|------|
| S | 서버 권위 (Server-Authoritative) - 클라이언트 값을 신뢰하지 않음 |
| V | 서버 검증 (Server-Validated) - 클라이언트 보고를 서버가 재현/검증 |
| M | 모니터링 (Monitoring) - 이상 패턴 감지 시 플래그 |
| C | 클라이언트 전용 (Client-Only) - 서버 검증 불필요 |

### 도메인별 치트 검증 요약 (14개 도메인)

| 도메인 | 권위 | API 수 | 핵심 검증 |
|--------|------|--------|-----------|
| 재화 (Resources) | S | 2 | 서버 잔액 관리, 트랜잭션 원장 |
| 재능 (Talent) | S | 3 | 레벨업 비용 서버 계산, 마일스톤 중복 수령 차단 |
| 장비 (Equipment) | S | 5 | 장비 UUID 서버 발급, 강화/합성/판매 비용 서버 검증 |
| 전투 (Battle) | V | 2 | SeededRandom 기반 전투 재현, 턴 수/속도핵 검증 |
| 챕터 (Chapter) | S+V | 8 | 일차 단방향 진행, 인카운터/스킬 선택 서버 관리 |
| 가챠 (Gacha) | S | 3 | 서버사이드 뽑기 실행, 천장 카운터 서버 관리 |
| 펫 (Pet) | S | 3 | 획득/등급업/배치 서버 관리 |
| 유산 (Heritage) | S | 2 | 해금 조건 서버 확인, 직업서 비용 차감 |
| 결제 (IAP) | S | 6 | 영수증 서버 검증, 중복 방지, 환불 감지 |
| 아레나 (PvP) | S+V | 6 | 서버 매칭, 전투 재현 검증, 포인트 서버 관리 |
| 일일 시스템 | S | 4 | 서버 UTC 시간 기준 리셋, 중복 수령 차단 |
| 컨텐츠 | S+V | 6 | 입장 횟수 서버 관리, 전투 결과 서버 실행/검증 |
| 세이브 | S | 3 | 서버 DB 정본, 단방향 진행, 체크섬 |
| 클라이언트 환경 | M | - | HTTPS + HMAC, 속도핵 타임스탬프 비교 |

### 단방향 진행 보장 (Monotonicity)

서버가 관리하는 모든 진행 필드는 역행 불가:

| 필드 | 방향 | 위반 시 |
|------|------|---------|
| Talent 레벨 | 증가만 | 거부 + 플래그 |
| 장비 강화 레벨 | 증가만 | 거부 |
| 타워 층수 | 증가만 | 거부 |
| ClearedChapterMax | 증가만 | 거부 + 플래그 |
| 펫 등급 | 승급만 | 거부 + 플래그 |

### 이상 감지 임계값

| 지표 | 정상 범위 | 플래그 임계값 | 자동 차단 |
|------|-----------|---------------|-----------|
| 시간당 골드 획득 | 0 ~ 50,000 | > 200,000 | > 1,000,000 |
| 시간당 전투 횟수 | 0 ~ 30 | > 60 | > 120 |
| 전투 평균 소요 시간 | 10초 ~ 5분 | < 3초 | < 1초 |
| 가챠 S등급 비율 | ~3% | > 20% (50회 이상) | > 50% (100회 이상) |

### 치트 플래그 자동 처리

| 누적 심각도 | 처리 |
|-------------|------|
| 1~5 | 경고 로그만 |
| 6~15 | 보상 미지급 (의심 전투) |
| 16~30 | 24시간 기능 제한 |
| 31+ | 계정 차단 (운영자 검토 후 해제) |
