# 서버 테스트 설계

## 작성일
2026-03-07

## 기획 의도
서버 비즈니스 로직의 정확성을 검증하기 위해 단위 테스트를 작성한다. 외부 의존성(DB, Redis, 스토어 API) 없이 서비스 로직만 검증하여 빠른 피드백 루프를 유지한다.

## 목표
- Server.Core 서비스 레이어 5개 서비스 100% 커버
- Mock 기반 단위 테스트 35개 작성
- 경계값, 예외 상황, 보안 시나리오 포함

## 동작 방식

### 기술 스택

| 항목 | 값 |
|------|-----|
| 테스트 프레임워크 | xUnit 2.9.2 |
| Mock 라이브러리 | NSubstitute 5.1.0 |
| 대상 | Server.Core 서비스 레이어 |
| 실행 환경 | .NET 8.0 |

### 테스트 범위

| 서비스 | 테스트 수 | 핵심 검증 |
|--------|----------|-----------|
| AuthService | 8 | 회원가입, 로그인, 차단 계정, JWT 토큰, RefreshToken |
| SaveService | 6 | 신규 저장, 클라이언트/서버 최신, 로드, 체크섬 |
| ShopService | 7 | 카탈로그 필터링, 구매 성공, 중복 영수증, 비활성 상품 |
| BattleVerifier | 7 | 전투 시작, 정상 보고, 계정 불일치, 시드 불일치, 속도핵 |
| ArenaService | 7 | 신규/기존 매칭, 자기 제외, 순위별 포인트 변동 |

### 테스트 명명 규칙

```
[메서드명]_[시나리오]_[기대결과]
```

### 실행 방법

```bash
cd Server
dotnet test
```

현재 환경에 .NET SDK가 설치되어 있지 않아 런타임 테스트 실행은 불가. CI 환경 또는 .NET 8.0 SDK 설치 후 실행 필요.
