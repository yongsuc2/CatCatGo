# 재능 강화 시 서버 연결 끊김

## 발생일
2026-03-07

## 환경
- Unity 클라이언트, 온라인 모드
- 관련 파일: GameManager.cs

## 증상
- 재능 강화를 반복하면 서버 연결이 끊어지고 오프라인 모드로 전환됨
- 골드가 부족한 상태에서 강화 시도를 몇 번 하면 재현됨

## 원인
`GameManager.OnApiFailed(string errorCode)`에서 서버의 **비즈니스 에러**(INSUFFICIENT_GOLD 등)를 **네트워크 실패**와 동일하게 처리.

- 서버가 `{ success: false, errorCode: "INSUFFICIENT_GOLD" }`를 정상 HTTP 200으로 반환
- 클라이언트 조건: `!response.Data.Success` → `OnApiFailed(response.Data.ErrorCode)` 호출
- `OnApiFailed`는 errorCode 유무와 무관하게 `_consecutiveFailures++` 실행
- `MaxConsecutiveFailures = 3` 도달 시 `SetNetworkMode(NetworkMode.OFFLINE)` 전환

즉, 골드 부족으로 3번 연속 실패하면 서버 연결이 끊긴 것으로 오판.

## 해결 방법
`OnApiFailed(string errorCode)`에서 분기 처리:
- `errorCode`가 있으면 (서버가 정상 응답한 비즈니스 에러) → `_consecutiveFailures = 0`으로 리셋, 토스트만 표시
- `errorCode`가 null이면 (실제 네트워크/HTTP 에러) → 기존대로 `_consecutiveFailures++`

## 추가 원인: 서버-클라이언트 비용 계산 불일치 (해결됨)
서버의 비용 계산이 클라이언트와 달라 INSUFFICIENT_GOLD 에러가 발생.

- 서버 (수정 전): `Math.Floor(100 * Math.Pow(1.10, tier))` — 단일 공식 하드코딩
- 클라이언트: `talent.data.json`의 gradeConfig별 baseCost/costGrowth 기반 구간 비용

서버 TalentService를 클라이언트와 동일한 등급별 비용 테이블 방식으로 수정하여 해결.
Grade별 baseCost: DISCIPLE=80, ADVENTURER=130, ELITE=420, MASTER=2600, WARRIOR=19000
CheckPromotion의 subGradeSums도 gradeConfig.tiers 누적합(2,7,17,34,67)으로 수정.

## 예방 방법
- API 실패 처리 시 "네트워크 실패"와 "비즈니스 로직 실패"를 명확히 구분
- `_consecutiveFailures`는 서버 응답을 받지 못한 경우에만 증가시킬 것
- 서버/클라이언트 간 비용·보상 등 게임 로직 수치는 동일한 데이터 소스(JSON)를 사용할 것
