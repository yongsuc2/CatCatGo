# 신규 계정 스태미나 부족 에러

## 발생일
2026-03-07

## 환경
- Unity 클라이언트 ↔ ASP.NET Core 서버 (Docker)
- 관련 파일: AuthService.cs, GameManager.cs, ResourceService.cs

## 증상
- 스태미나가 충분히 있는데 "스태미나가 부족합니다" 메시지 표시
- 계정 탈퇴 → 재가입 후 모험 시작 시 특히 발생

## 원인
서버와 클라이언트의 신규 계정 초기 리소스 불일치.

- 클라이언트 `GameManager.InitNewGame()` (line 171-179): 새 게임 시작 시 GOLD=500, GEMS=500, STAMINA=100으로 초기화
- 서버 `AuthService.RegisterAsync()`: 계정 생성만 하고 **리소스 초기화 없음** → 잔액 0
- 클라이언트는 스태미나 100이 있다고 UI에 표시하지만, 서버에서 `ResourceService.SpendAsync`로 스태미나 5 차감 시도 → 잔액 0 < 5 → `INSUFFICIENT_STAMINA` 반환

재능 비용 불일치(TalentService)와 동일한 패턴: 서버-클라이언트 간 게임 데이터 초기화 로직이 분리되어 불일치 발생.

## 해결 방법
`AuthService.RegisterAsync()`에서 계정 생성 직후 `ResourceService.GrantMultipleAsync`로 초기 리소스 지급:
- GOLD: 500
- GEMS: 500
- STAMINA: 100

`Program.cs`에서 `AuthService` 생성 시 `ResourceService` 의존성 주입 추가.

## 미해결 사항
- `ResetDataAsync()` (계정 리셋) 시에도 초기 리소스 재지급이 필요할 수 있음 — 현재 미확인
- 다른 리소스(토큰류 등)의 초기값이 있는지 확인 필요

## 예방 방법
- 서버-클라이언트 간 게임 초기화 로직은 반드시 동일한 데이터를 참조할 것
- 신규 기능 추가 시 "서버에도 동일한 초기값/비용이 설정되었는가" 체크리스트 필수
