### 서버 연동 에러 수정

#### 수행 작업
- `ApiClient.PingServer` 응답 판정 수정: `ProtocolError`(HTTP 4xx/5xx)도 서버 온라인으로 판정하도록 변경
- `SaveApi.Load()` 404 처리 확인: 신규 계정(세이브 없음)의 404 응답이 에러 로그 없이 정상 처리되는 것 확인

#### 주요 변경사항
- `Assets/_Project/Scripts/Network/ApiClient.cs`: PingServer 콜백 조건에 `UnityWebRequest.Result.ProtocolError` 추가

#### 분석 내용
- PingServer는 `UnityWebRequest.Head(BaseUrl)`로 서버에 HEAD 요청을 보냄
- 서버가 해당 경로에 대해 404를 반환해도 `ProtocolError`로 분류되어 IsOnline=false가 되었음
- ProtocolError는 HTTP 응답이 온 것이므로 서버가 살아있다는 의미 -> IsOnline=true가 정확
- SaveApi.Load 404 처리: `ServerSyncService.TryLoadServerSave()`에서 `!response.IsSuccess` 체크 후 조용히 return하므로 수정 불필요
