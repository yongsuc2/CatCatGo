# 서버 아키텍처

## 작성일
2026-03-07

## 기획 의도
CatCatGo는 결제(IAP), 치트 방지, 클라우드 세이브, 비동기 PvP 등 서버가 필수적인 기능을 포함한다. 클라이언트 단독으로는 영수증 위변조 방지, 계정 복구, 일일 리셋 통일 등이 불가능하므로 서버 권위 모델(Server-Authoritative)을 도입한다.

## 목표
- 결제/재화/세이브/전투 결과에 대한 서버 권위 모델 수립
- 도메인 로직(Domain, Infrastructure)을 클라이언트-서버 간 공유
- 오프라인 폴백을 지원하는 듀얼 모드 아키텍처

## 동작 방식

### 기술 스택

| 항목 | 값 |
|------|-----|
| 서버 프레임워크 | ASP.NET Core 8.0 |
| 데이터베이스 | PostgreSQL 16 |
| 캐시/세션 | Redis 7 |
| 인증 | JWT + Refresh Token |
| 프로토콜 | REST API (HTTPS) |
| 컨테이너 | Docker Compose |

### 서버 권위 모델

| 영역 | 권위 | 설명 |
|------|------|------|
| 재화 변동 | 서버 | 골드/보석/스태미나 증감은 서버만 가능 |
| 결제 | 서버 | 스토어 영수증 서버 검증 후 지급 |
| 세이브 | 서버 | 서버 DB가 정본, 클라이언트는 캐시 |
| 전투 결과 | 클라이언트 주도 + 서버 검증 | 클라이언트가 결과 보고 -> 서버가 시드로 재현 검증 |
| 아레나 | 서버 | 매칭/랭킹/보상 서버 관리 |
| 일일 리셋 | 서버 | UTC 기준 서버 시간 사용 |
| 가챠 | 서버 | 확률 조작 방지, 천장 카운터 서버 관리 |

### 도메인 로직 공유

```
Unity Client                    Game Server
+---------------+              +---------------+
| Presentation  |              |  Server.Api   |
| Services      |              |  Server.Core  |
| +-----------+ |              | +-----------+ |
| |  Domain   |<+---- 동일 ---->| |  Domain   | |
| |  Infra    | |   어셈블리    | |  Infra    | |
| +-----------+ |              | +-----------+ |
+---------------+              +---------------+
```

- Domain, Infrastructure 어셈블리는 Unity 비의존 -> 서버에서 그대로 참조
- SeededRandom으로 전투 결과를 서버에서 독립 재현 -> 치트 검증
