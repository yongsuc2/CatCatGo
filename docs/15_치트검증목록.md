# 치트 검증 목록 + 서버 구현 항목

## 범례

| 기호 | 의미 |
|------|------|
| **S** | 서버 권위 (Server-Authoritative) — 클라이언트 값을 신뢰하지 않음 |
| **V** | 서버 검증 (Server-Validated) — 클라이언트 보고를 서버가 재현/검증 |
| **M** | 모니터링 (Monitoring) — 이상 패턴 감지 시 플래그 |
| **C** | 클라이언트 전용 (Client-Only) — 서버 검증 불필요 |

---

## 1. 재화 (Resources) — 모든 재화는 **S**

클라이언트의 재화 값을 절대 신뢰하지 않음. 서버 DB가 정본.

| 재화 | 치트 벡터 | 서버 구현 |
|------|-----------|-----------|
| **골드 (GOLD)** | 메모리 에딧, 세이브 파일 조작 | 서버가 잔액 관리. 증감 API만 노출. 모든 획득/소비를 트랜잭션 로그 |
| **보석 (GEMS)** | 메모리 에딧, 결제 위변조 | 서버가 잔액 관리. 결제 영수증 검증 후만 지급. 무료 보석도 서버 기록 |
| **스태미나 (STAMINA)** | 시간 조작, 무한 스태미나 | 서버 시간 기준 회복 계산. 최대치(100) 캡 서버 적용. 클라이언트 시간 무시 |
| **도전권 (CHALLENGE_TOKEN)** | 일일 리셋 악용, 수량 조작 | 서버 일일 리셋 시 지급. 사용 시 서버 차감 |
| **곡괭이 (PICKAXE)** | 일일 리셋 악용 | 서버 일일 리셋 시 지급. 사용 시 서버 차감 |
| **장비 강화석 (EQUIPMENT_STONE)** | 수량 조작 | 획득처(던전/뽑기/타워) 모두 서버 지급 |
| **도면 (POWER_STONE)** | 수량 조작 | 타워 보상 서버 지급 |
| **직업서 4종 (SKULL/KNIGHT/RANGER/GHOST_BOOK)** | 수량 조작 | 컨텐츠 보상 서버 지급 |
| **펫 알 (PET_EGG)** | 수량 조작 | 가챠 보상 서버 지급 |
| **펫 먹이 (PET_FOOD)** | 수량 조작 | 가챠 보상 서버 지급 |

### 서버 구현: ResourceService
```
POST /api/resource/spend     ← 재화 소비 (서버 잔액 확인 후 차감)
GET  /api/resource/balance   ← 현재 잔액 조회
(내부) ResourceLedger        ← 모든 증감 트랜잭션 로그 (감사 추적)
```

## 2. 재능 (Talent) — **S**

| 치트 벡터 | 상세 | 서버 구현 |
|-----------|------|-----------|
| 스탯 직접 조작 | HP/ATK/DEF 레벨을 비정상 값으로 설정 | 레벨업 API만 노출. 서버가 비용 확인 → 차감 → 레벨 증가 |
| 비용 우회 | 골드 없이 레벨업 | costGrowth(1.10) × baseCost 공식을 서버에서 계산하여 비용 검증 |
| 등급 조작 | TalentGrade를 직접 HERO로 설정 | 서브등급 진급 조건(10레벨 도달) 서버 검증 |
| 마일스톤 보상 중복 수령 | ClaimedMilestones 조작 | 서버가 수령 기록 관리. 동일 키 중복 수령 차단 |
| 골드 획득량 부스트 조작 | GetGoldMultiplier() 값 변조 | 서버가 마일스톤 기반으로 직접 계산 |

### 서버 구현: TalentService
```
POST /api/talent/upgrade         ← 스탯 레벨업 (type: HP/ATK/DEF, 서버 비용 계산+차감)
POST /api/talent/claim-milestone ← 마일스톤 보상 수령 (중복 방지)
GET  /api/talent/status          ← 현재 등급/레벨/서브등급 조회
```

## 3. 장비 (Equipment) — **S**

| 치트 벡터 | 상세 | 서버 구현 |
|-----------|------|-----------|
| 장비 생성 | 존재하지 않는 장비를 인벤토리에 추가 | 장비 획득(뽑기/합성/전투 보상)은 서버만 실행. 장비 UUID 서버 발급 |
| 등급 조작 | COMMON → MYTHIC 변경 | 장비 데이터 서버 DB에 저장. 클라이언트 수정 무시 |
| 부스탯 조작 | SubStat 값 변조, 개수 초과 | 장비 생성 시 서버가 SubStat 생성 (등급별 0~5개, 슬롯별 후보 풀) |
| 강화 레벨 조작 | enhancementLevel 직접 증가 | 강화 API: 서버가 비용(EQUIPMENT_STONE+GOLD) 확인 → 차감 → 레벨 증가 |
| S등급 플래그 조작 | IsS를 true로 설정 | 가챠 시 서버가 sRate 확률로 판정, 결과에 포함 |
| 합성(Forge) 조작 | 재료 없이 합성, 결과 등급 조작 | 합성 API: 서버가 재료 장비 존재+등급 확인 → 결과 장비 생성 |
| 판매 가격 조작 | SellEquipment에서 가격 변조 | 판매 API: 서버가 EquipmentDataTable.GetSellPrice로 가격 계산 |
| 장착 슬롯 초과 | MaxCount(3) 이상 장착 | 장착 API: 서버가 슬롯 상태 확인 |

### 서버 구현: EquipmentService
```
POST /api/equipment/enhance     ← 강화 (비용 차감 + 레벨업)
POST /api/equipment/forge       ← 합성 (재료 소멸 + 결과 생성)
POST /api/equipment/equip       ← 장착 (슬롯 검증)
POST /api/equipment/unequip     ← 해제
POST /api/equipment/sell        ← 판매 (서버 가격 계산)
```

## 4. 전투 (Battle) — **V**

| 치트 벡터 | 상세 | 서버 구현 |
|-----------|------|-----------|
| 전투 결과 조작 | 패배를 승리로 보고 | 시드 기반 전투 재현 — SeededRandom으로 서버가 독립 실행, 결과 비교 |
| 보상 뻥튀기 | 골드/아이템 보상 변조 | 서버가 전투 결과에 따라 보상 계산+지급 |
| 턴 수 조작 | 적은 턴으로 클리어 보고 | 재현 시 턴 수 비교. 허용 오차 ±1턴 |
| 속도핵 | 1초 만에 전투 완료 | 전투 소요 시간 vs 턴 수 × 최소 시간 비교. `elapsed < turns × 0.5s` 시 플래그 |
| 스킬 조작 | 보유하지 않은 스킬 사용 | 서버가 세션 스킬 목록 관리. 전투 시 보유 스킬만 허용 |
| 적 데이터 조작 | 약한 적으로 교체 | 서버가 챕터/일차/인카운터에 맞는 적 선택 (EnemyTable + 테마 풀) |
| 원킬핵 | 데미지 값 변조 | 재현 시 총 데미지 비교. 유의미한 차이 시 플래그 |

### 서버 구현: BattleVerifier (확장)
```
POST /api/battle/start    ← 전투 시작 → 서버가 시드 + 적 정보 + 세션 스킬 스냅샷 저장
POST /api/battle/report   ← 전투 결과 보고 → 서버 재현 검증 → 보상 확정/거부
```

### 전투 재현에 필요한 서버 데이터
- 플레이어 스탯 스냅샷 (ComputeStats 결과)
- 세션 스킬 목록 (챕터 내 획득한 스킬들)
- 장비 패시브 스킬 목록
- 적 템플릿 ID + 챕터 스케일링 계수
- SeededRandom 시드
- 밸런스 데이터 (battle.data.json — 서버/클라이언트 동일 버전)

## 5. 챕터/인카운터 (Chapter) — **V + S**

| 치트 벡터 | 상세 | 서버 구현 |
|-----------|------|-----------|
| 일차 건너뛰기 | day를 직접 60일로 설정 | 서버가 챕터 진행 상태(currentDay) 관리. 인카운터 완료 시만 day+1 |
| 인카운터 결과 조작 | 악마 → 보상만 취득 | 서버가 인카운터 타입별 결과 검증 (EncounterGenerator + 시드) |
| 스킬 획득 조작 | 챕터 내 미획득 스킬 추가 | 스킬 획득 인카운터는 서버가 후보 3개 생성 → 클라이언트가 선택 → 서버 확인 |
| 스킬 리롤 무한 사용 | rerollCount 초기화 | 서버가 세션당 리롤 횟수(2회) 관리 |
| 중박/대박 카운터 조작 | 카운터를 임계값으로 설정 | 서버가 카운터 증가/리셋 관리 |
| 챕터 클리어 위조 | 미클리어를 클리어로 보고 | 60일차 보스 전투 승리를 서버가 확인 |
| 챕터 보물 중복 수령 | 마일스톤 보상 재수령 | 서버가 수령 기록(BestSurvivalDays) 관리 |

### 서버 구현: ChapterService
```
POST /api/chapter/start           ← 챕터 시작 (스태미나 5 차감, 시드 발급)
POST /api/chapter/encounter       ← 인카운터 선택 → 서버가 결과 생성
POST /api/chapter/encounter/resolve ← 인카운터 해결 (선택지 적용)
POST /api/chapter/skill/select    ← 스킬 선택 (서버가 후보 검증)
POST /api/chapter/skill/reroll    ← 스킬 리롤 (횟수 차감)
POST /api/chapter/abandon         ← 챕터 포기
GET  /api/chapter/state           ← 현재 챕터 상태 조회
POST /api/chapter/treasure/claim  ← 챕터 보물 수령
```

## 6. 가챠/뽑기 (Gacha) — **S**

| 치트 벡터 | 상세 | 서버 구현 |
|-----------|------|-----------|
| 확률 조작 | 무조건 MYTHIC 획득 | 서버에서 뽑기 실행 (TreasureChest 로직 서버 실행) |
| 천장 카운터 조작 | pityCount를 임계값으로 설정 | 서버가 천장 카운터 관리 |
| 비용 우회 | 보석 없이 뽑기 | 서버가 보석 잔액 확인 → 차감 → 뽑기 실행 |
| S등급 조작 | sRate 확률 변조 | 서버가 sRate로 판정 |
| 10연차 할인 악용 | 1회 비용으로 10회 실행 | 서버가 10연차 비용(9회분) 계산+차감 |

### 서버 구현: GachaService (기존 확장)
```
POST /api/gacha/pull    ← 1회 뽑기 (서버 실행, 비용 차감, 천장 증가, 결과 반환)
POST /api/gacha/pull10  ← 10연차 (서버 실행)
GET  /api/gacha/pity    ← 천장 카운터 조회
```

| 치트 벡터 | 상세 | 서버 구현 |

### 서버 구현: PetService
```
POST /api/pet/feed     ← 먹이주기 (비용 차감 + 경험치 증가)
POST /api/pet/upgrade  ← 등급업 (조건 검증)
POST /api/pet/equip    ← 활성 펫 변경
```

## 9. 결제 (IAP) — **S**

| 치트 벡터 | 상세 | 서버 구현 |
|-----------|------|-----------|
| 영수증 위조 | 가짜 영수증으로 보석 획득 | Google Play / App Store 서버 API로 영수증 검증 |
| 영수증 재사용 | 동일 영수증 반복 제출 | receipt_id UNIQUE 제약 (DB) |
| 환불 악용 | 결제 후 환불, 보석 유지 | RTDN/S2S 알림으로 환불 감지 → 보석 회수 |
| 상품 가격 조작 | 클라이언트에서 가격 변조 | 서버가 상품 테이블에서 가격 조회 (클라이언트 가격 무시) |
| 이벤트 패키지 기한 악용 | 만료된 패키지 구매 시도 | 서버가 start_at/end_at 확인 |

### 서버 구현: ShopService (기존 확장)
```
POST /api/shop/purchase          ← 결제 영수증 제출 → 검증 → 지급
POST /api/shop/consume           ← 소모성 상품 사용
GET  /api/shop/catalog           ← 현재 활성 상품 목록 (이벤트 포함)
GET  /api/shop/subscription      ← 월정액 상태
webhook /api/shop/rtdn           ← Google RTDN 콜백
webhook /api/shop/s2s-notification ← Apple S2S 콜백
```

## 10. 아레나 (PvP) — **S + V**

| 치트 벡터 | 상세 | 서버 구현 |
|-----------|------|-----------|
| 매칭 조작 | 약한 상대만 선택 | 서버가 같은 티어에서 랜덤 매칭 |
| 전투 결과 조작 | 패배를 승리로 보고 | 아레나 전투도 시드 기반 재현 검증 |
| 티어 포인트 조작 | 포인트 직접 증가 | 서버가 포인트 관리 (전투 결과 기반) |
| 입장권 무한 사용 | ENTRY_PASS 차감 회피 | 서버가 입장권 차감 |
| 재도전 비용 우회 | 보석 없이 재도전 | 서버가 보석 차감 확인 |
| 스냅샷 조작 | 방어 덱 스탯 뻥튀기 | 서버가 스탯 직접 계산 (Player.ComputeStats 서버 실행) |

### 서버 구현: ArenaService (기존 확장)
```
POST /api/arena/match       ← 매칭 요청 (입장권 차감, 상대 4명 선택)
POST /api/arena/result      ← 전투 결과 (서버 검증, 포인트 변동)
POST /api/arena/defense     ← 방어 덱 업데이트 (서버가 스탯 스냅샷 계산)
GET  /api/arena/ranking     ← 랭킹 조회
GET  /api/arena/season      ← 시즌 정보
POST /api/arena/retry       ← 재도전 (보석 차감)
```

|-----------|------|-----------|
| 시간 조작 | 기기 시간 변경으로 일일 리셋 반복 | 서버 UTC 시간 기준 리셋. 클라이언트 시간 무시 |
| 출석 보상 중복 수령 | 동일 날짜 반복 수령 | 서버가 수령 날짜 기록 관리 |
| 퀘스트 진행도 조작 | 퀘스트 완료 위조 | 서버가 액션 발생 시 퀘스트 진행도 증가 |
| 퀘스트 보상 중복 수령 | 완료된 퀘스트 재수령 | 서버가 수령 기록 관리 |
| 일일 던전 재입장 | 무료 횟수 초과 | 서버가 일일 입장 횟수 관리 |

### 서버 구현: DailyService
```
POST /api/daily/attendance/claim   ← 출석 보상 수령
GET  /api/daily/attendance         ← 출석 현황
POST /api/daily/quest/claim        ← 퀘스트 보상 수령
GET  /api/daily/quest              ← 퀘스트 진행도
(내부) DailyResetJob               ← UTC 0시 크론잡 — 도전권/곡괭이 리셋, 퀘스트 리셋
```

## 14. 클라이언트 환경 — **M**

| 치트 벡터 | 상세 | 서버 구현 |
|-----------|------|-----------|
| 메모리 에디터 | GameGuardian/Cheat Engine으로 런타임 값 변경 | 서버 권위 모델이면 무효화됨 (서버 값이 정본) |
| 패킷 변조 | MITM으로 API 요청/응답 수정 | HTTPS + 요청 서명(HMAC) + 서버사이드 검증 |
| 에뮬레이터 | 루팅된 환경에서 실행 | (선택) SafetyNet/Play Integrity API |
| 앱 변조 | APK 디컴파일 후 수정 | (선택) 코드 난독화 + 무결성 검증 |
| 속도핵 | 게임 속도 가속 | 서버 타임스탬프 비교. 비현실적 속도 시 플래그 |
| 자동화 봇 | 반복 작업 자동화 | 행동 패턴 분석 (일정한 간격, 24시간 연속 접속) |

### 서버 구현: AntiCheatService
```
(미들웨어) RequestSignatureValidator  ← HMAC 서명 검증
(미들웨어) RateLimiter                ← API 요청 빈도 제한
(크론잡)   BehaviorAnalyzer           ← 비정상 패턴 감지 (속도핵, 봇)
(DB)       cheat_flags 테이블         ← 의심 플래그 누적 → 자동 차단 임계값
```
