# 카피바라고! 역기획서 - 전투 시스템

## 전투 방식

| 항목 | 내용 |
|------|------|
| 전투 유형 | 턴제 자동 전투 |
| 선공 | 플레이어 선공 |
| 전투 진행 | 자동 (플레이어 개입 없음) |
| 전투 결과 | 승리 시 보상 획득, 패배 시 챕터 종료 |

## 전투 흐름

```
[전투 시작]
    ↓
[플레이어 턴 스킬] → 턴 시작 트리거 스킬 발동
    ↓
[적 턴 스킬] → 적 턴 시작 트리거 스킬 발동
    ↓
[플레이어 공격 애니메이션]
    접근(350ms) → 타격 + 데미지 팝업(300ms) → 후퇴(350ms)
    ↓
[적 공격 애니메이션]
    접근(350ms) → 타격 + 데미지 팝업(300ms) → 후퇴(350ms)
    ↓
[턴 종료] → 턴 기반 효과 처리 (DoT, HoT, 버프/디버프)
    ↓
[승패 판정] → HP 0 이하 시 패배/승리
    ↓
[다음 턴 또는 전투 종료]
```

## 분노(Rage) 시스템

모든 유닛(플레이어/적)이 동일한 전투 로직과 스킬 시스템을 사용한다. 빌트인 스킬(ilban_attack, bunno_attack, rage_accumulate)은 모든 유닛에 자동 장착된다. 차이는 데이터(스킬 구성)뿐이다.

| 항목 | 내용 |
|------|------|
| 분노 게이지 최대치 | `BattleDataTable.rage.maxRage` (100) |
| 분노 게이지 상승량 | 캐릭터 스탯 `ragePerAttack` (플레이어: rage_accumulate 티어 데이터, 적: `enemy.data.json`) |
| 분노 충전 방식 | 일반 공격 시 rage_accumulate 스킬이 `ragePerAttack` 스탯만큼 충전 |
| 분노 공격 | 물리 공격, 스킬 계수(bunno_attack 티어 데이터)로 데미지 결정 |
| ON_RAGE 스킬 | 분노 공격 발동 시 추가 스킬 자동 발동 |

### 분노 관련 스킬

| 스킬 | 효과 |
|------|------|
| rage_accumulate (분노 축적) | 일반 공격 시 `ragePerAttack` 만큼 분노 게이지 충전 |
| bunno_attack (분노 공격) | 분노 100 소모 + 물리 공격 (계수는 `active-skill-tier.data.json` 참조) |
| rage_gauge_boost (분노 게이지 증가) | 일반 공격 시 추가 분노 게이지 충전 (고정 수치) |
| rage_mastery (분노 마스터리) | 특정 스킬 ID의 데미지 증가 (SKILL_MODIFIER) |
| bunno_thunder / bunno_lance / bunno_flame | 분노 공격 시 추가 스킬 발동 |

### 전투 흐름 (분노 포함)

```
[유닛 공격]
  ├─ 분노 게이지 ≥ 최대치?
  │   ├─ YES → 분노 공격 (CONSUME_RAGE 100 + PHYSICAL ATTACK × 스킬 계수)
  │   │        ├─ ON_RAGE 스킬 발동 (번개/광창/화염파)
  │   │        └─ 분노 게이지 -100 (초과분 유지)
  │   └─ NO → 일반 공격
  │            ├─ 흡혈 처리
  │            ├─ UPPER 스킬 발동
  │            ├─ 분노 게이지 충전 (rage_accumulate → ragePerAttack 스탯 사용)
  │            ├─ 연타 판정 → 성공 시:
  │            │    ├─ 일반 공격 한 번 더 (연타 재발동 없음)
  │            │    └─ UPPER 스킬 다시 발동
  │            └─ IF 분노 게이지 ≥ 최대치 (즉시 발동):
  │                 ├─ 분노 공격 (같은 턴에 연속)
  │                 ├─ ON_RAGE 스킬 발동
  │                 └─ 분노 게이지 -100 (초과분 유지)
  └─ 반격 체크
```

### 전투 연출 시스템

#### 기본 규칙
- 동시 공격 없음: 플레이어 공격 완료 후 적 공격
- 최대 턴: `BattleDataTable.maxTurns` (초과 시 패배 처리)
- 배속: 1x/2x 토글 (전투 중 변경 가능, CSS `--battle-speed` 변수 + JS delay 분할)

#### 페이즈 타이밍

| 페이즈 | 기본 시간 | 설명 |
|--------|----------|------|
| approach | 350ms | 공격자가 피격자 방향으로 이동 (투사체가 있으면 아이콘이 날아감) |
| hit | 300ms | 타격 시점 — 데미지 팝업 표시 + HP바 갱신 + 피격자 흔들림(shake) |
| retreat | 350ms | 공격자 원위치 복귀 |
| pause | 200ms | 다음 행동 전 대기 |

배속 적용: 모든 시간 ÷ `speedMultiplier`

#### 턴 1회 연출 순서

```
animateTurn()
│
├─ 1. executeTurn() — 도메인 로직 1턴 전체 실행 (결과 로그 수집)
│
├─ 2. 로그 엔트리 분류
│     ├─ playerEntries: source = 플레이어 (공격/스킬/반격/분노/흡혈)
│     ├─ enemyEntriesBySource: source = 각 적 (공격/스킬/반격/분노/흡혈)
│     └─ statusEntries: DOT_DAMAGE / HOT_HEAL (상태효과 틱)
│
├─ 3. 딜 그래프 누적 (accumulateDamage)
│     ├─ 피해: 스킬명별 집계 (일반 공격/반격/분노 공격/독 피해 등)
│     └─ 회복: 유형별 집계 (흡혈/재생/부활)
│
├─ 4. 초기 HP 역산
│     └─ 최종 HP에서 모든 엔트리의 델타를 빼서 "턴 시작 시점 HP" 복원
│
├─ 5. 플레이어 공격 애니메이션
│     ├─ 히트 그룹별 순차 재생 (splitToAnimationGroups)
│     │   ├─ 일반/스킬/크리티컬 → 개별 그룹
│     │   ├─ 분노 공격 → 개별 그룹 (일반 공격으로 충전 시 같은 턴에 연속)
│     │   └─ 흡혈(LIFESTEAL) → 직전 그룹에 합산
│     └─ 각 그룹: approach → hit(데미지 팝업 + HP 갱신) → retreat → pause
│
├─ 6. 적 공격 애니메이션 (적별 순차)
│     ├─ 복수 적일 때 각 적이 순서대로 공격
│     ├─ 히트 그룹별 순차 재생 (플레이어와 동일 구조)
│     └─ 플레이어 HP ≤ 0 이면 즉시 중단
│
├─ 7. 상태효과 틱 연출
│     ├─ 캐릭터 이동 없음 (attackPhase = 'idle')
│     ├─ 데미지 팝업만 표시 (hit 시간 동안)
│     └─ HP바 갱신 (DoT 피해 감소 / HoT 회복 증가)
│
└─ 8. 최종 상태 동기화
      └─ setPlayerUnit / setEnemyUnits를 도메인 객체 최종 상태로 덮어씀
```

#### 히트 그룹 분할 규칙 (splitToAnimationGroups)

한 사이드(플레이어 or 적)의 로그 엔트리를 접근-타격-후퇴 단위로 묶는다.

**1단계: 같은 스킬끼리 정렬 (reorderBySkillType)**

UPPER 스킬이 여러 개 있으면 서로 다른 UPPER 스킬이 같은 자식 스킬을 호출하여 엔트리가 교차될 수 있다. 이를 같은 스킬끼리 묶어 연속 연출한다.

- 배치 경계: ATTACK, RAGE_ATTACK, 일반 공격 CRIT
- 배치 내 SKILL_DAMAGE/CRIT 엔트리를 `skillName` 기준으로 그룹화 (첫 등장 순서 유지)
- LIFESTEAL 엔트리는 배치 끝에 배치

```
변경 전: 일반 → 수리검 → 번개 → 수리검 → 번개 → 번개
변경 후: 일반 → 수리검 → 수리검 → 번개 → 번개 → 번개
```

multi-hit 발동 시 일반 공격이 새로운 배치를 만들어 별도 연출:

```
배치 1: 일반 → 수리검×2 → 번개×3
배치 2: 일반(연타) → 수리검 → 번개
```

**2단계: 그룹 생성**

| 엔트리 타입 | 그룹핑 |
|------------|--------|
| ATTACK, SKILL_DAMAGE, CRIT, COUNTER | 각각 새 그룹 |
| RAGE_ATTACK | 새 그룹 |
| LIFESTEAL | 직전 그룹에 합산 (공격 데미지와 흡혈을 한 번에 표시) |

#### HP 중간 갱신 (computeMidTurnHp)

한 히트 그룹의 데미지/힐 엔트리를 순차 적용하여 중간 HP를 계산한다.
- 피해 엔트리: target의 HP 감소
- 힐 엔트리 (LIFESTEAL, HOT_HEAL, REVIVE): target의 HP 증가
- 결과 HP는 `[0, maxHp]` 범위로 클램프

#### 투사체 이펙트
- 스킬 데미지에 `skillIcon`이 있으면 캐릭터 이동 대신 아이콘이 대상에게 날아감
- SKILL_DAMAGE와 CRIT(일반 공격 제외) 모두 투사체 대상
- 일반 공격/분노 공격/반격은 캐릭터 이동 방식 유지
- 투사체 그룹은 approach 시작 전에 damageEntries를 설정하여 approach 동안 날아감

### 데미지 팝업 매핑

| 로그 타입 | 표시 | 색상 |
|-----------|------|------|
| ATTACK, DOT_DAMAGE | -{값} | 빨강 (#ff5252) |
| SKILL_DAMAGE | {스킬아이콘} -{값} | 빨강 (#ff5252) |
| CRIT | CRIT! {아이콘} -{값} | 노랑 (#ffd740), 24px, 글로우 |
| COUNTER | -{값} | 빨강 (#ff5252) |
| RAGE_ATTACK | -{값} | 주황 (#ff6b00), 글로우 |
| LIFESTEAL, HOT_HEAL, REVIVE | +{값} | 초록 (#69f0ae) |

팝업 표시 시간: 600ms ÷ speedMultiplier

### 상태 효과 아이콘

캐릭터 HP바/분노바 아래에 현재 적용 중인 상태 효과를 아이콘으로 표시한다.

| 상태 | 아이콘 | 분류 | 중첩 |
|------|--------|------|------|
| POISON (독) | 🧪 | debuff | 중첩 가능 (×N 표시) |
| BURN (화상) | 🔥 | debuff | 중첩 가능 (×N 표시) |
| REGEN (재생) | 💚 | buff | 단일 |
| ATK_UP | ⚔️ | buff | 단일 |
| ATK_DOWN | ⚔️ | debuff | 단일 |
| DEF_UP | 🛡️ | buff | 단일 |
| DEF_DOWN | 🛡️ | debuff | 단일 |
| CRIT_UP | 🎯 | buff | 단일 |

- buff: 초록 테두리, debuff: 빨간 테두리
- 우측 하단에 남은 턴 수 표시
- DoT 중첩 규칙:
  - 같은 스킬(sourceSkillId 동일) → 상위 등급(높은 value)이 덮어씀, 같은 등급이면 재적용(턴 리프레시)
  - 다른 스킬(예: poison_inject + flame_summon) → 공존, 각각 독립 표시

## 스탯 체계

| 스탯 | 설명 | 강화 경로 |
|------|------|-----------|
| HP | 최대 체력 | 특성, 장비(의상/장신구), 스킬 |

## 스킬 발동 구조

### 스킬 등급

| 등급 | 설명 |

| 유형 | 예시 | 효과 |
|------|------|------|
| 직접 공격 | 번개, 광창, 수리검, 검기 | 추가 데미지 |
| 반격 | 반격 계열 | 피격 시 자동 반격 |
| 연타 | 연타 마스터리 | 일반 공격 시 확률로 추가 타격 |
| 치명타 | 치명타 마스터리 | 크리티컬 확률/배율 상승 |
| 흡혈 | 흡혈 계열 | 일반 공격 피해의 일부 HP 회복 (스킬 데미지 미적용) |

| 분노 | 분노 마스터리 | 분노 게이지 → 분노 공격 자동 발동 |

## 복수 적 전투 (1 vs 2)

일반 전투에서 확률적으로 2마리가 동시 등장한다. 엘리트/중간보스/최종 보스는 항상 1마리.

| 항목 | 내용 |
|------|------|
| 출현 확률 / 스탯 배율 | `BattleDataTable.enemy` 참조 |
| 적 구성 | 서로 다른 종류 2마리 |
| 타겟팅 | 순차 공격 (앞 적부터 처치, 죽으면 다음) |
| 개별 처리 | 각 적이 독립적으로 공격/스킬/분노/상태효과 |

### 전투 흐름 (복수 적)

```
executeTurn()
├── 턴 시작 스킬: 모든 살아있는 적 ↔ 플레이어
├── 플레이어 공격: 첫 번째 살아있는 적 순차 공격
├── 각 적 순차 공격: 살아있는 적마다 플레이어 공격
│   └── 플레이어 사망 시 즉시 중단
├── 상태 효과 틱: 플레이어 + 모든 적
└── 승패 판정: 모든 적 사망 = VICTORY, 플레이어 사망 = DEFEAT
```

## 방어막(Shield) 시스템

| 항목 | 내용 |
|------|------|
| 획득 방법 | PASSIVE 스킬 (iron_shield 등) |
| 데미지 흡수 | 쉴드가 먼저 깎이고, 남은 데미지가 HP에 적용 |
| 방어막량 | maxHp × skill.effect.value (`SkillTable` 참조) |
| 표시 | HP 바 안에 거의 흰색으로 HP 게이지 뒤에 이어서 표시. HP+쉴드 > maxHP 시 HP 위에 겹침 (바 크기 고정, 쉴드 항상 전체 표시) |

## 몬스터 컨셉/스킬 배치

10챕터 단위 5개 테마로 구성. 미생물 → 곤충 → 소형 동물 → 중형 동물 → 대형 동물.
전체 적 목록 및 테마 구성: `docs/12_모험시스템.md` 참조. 스탯 상세: `enemy.data.json` 참조.

### 테마 1: 미생물 (챕터 1~10)

| 몬스터 | 컨셉 | 스킬 |

## 데미지 공식

모든 계산은 float으로 수행하며, 방어력 감소를 최종 단계에 적용한 뒤 **1회만 정수 절삭**한다.

```
물리: (ATK + hpBonus) × 스킬계수 × 마스터리 × 크리티컬 × k/(k+DEF)
마법: ATK × 마법계수 × 스킬계수 × 마스터리 × k_m/(k_m+DEF)
고정: ATK × 스킬계수 × 마스터리
```

| 요소 | 설명 |
|------|------|
| 스킬계수 | 스킬 데이터의 `effect.coefficient` |
| 마스터리 | `1 + masteryBonus` (SKILL_MODIFIER 패시브, 없으면 1.0) |
| 크리티컬 | 물리만 적용. 확률 판정 성공 시 `BattleDataTable.damage.critMultiplier` |
| 방어력 감소 | `k / (k + DEF)` — 최종 단계에 적용 |
| hpBonus | MAX_HP_DAMAGE 패시브 (maxHp × coefficient) |

## 전투 계수 데이터 테이블

모든 전투 관련 계수는 `src/domain/data/BattleDataTable.ts`에서 중앙 관리한다.

| 카테고리 | 관리 항목 |
|---------|---------|
| damage | 방어력 감소 비율, 크리티컬 배율 (물리 전용) |
| rage | 최대 게이지 (maxRage) |
| enemy | 2마리 출현 확률/스탯 배율, 챕터/탑 스케일링, 일수 기반 보너스 |
| combatGoldReward | 일반 전투 골드: base + perChapter × chapterId + perDay × currentDay |
| maxTurns | 전투 최대 턴 수 |

## 전투 난이도 요소

| 요소 | 설명 |
|------|------|
| 적 체력/공격력 | 챕터가 높을수록 증가, 챕터 내 일수 진행에 따라 최대 +80% 보너스 |
| 적 스킬 | 고난이도 적은 특수 스킬 보유 |
| 보스 | 챕터 마지막에 등장, 높은 스탯 + 특수 패턴 |
| 엘리트 | 일반 적보다 강화된 중간 보스급 |
| 복수 적 | 일반 전투에서 확률적으로 2마리 동시 등장 (스탯 감소) |
| 방어막 | 일부 적이 방어막 보유 (데미지 흡수) |
| 분노 공격 | 모든 유닛이 분노 게이지 보유, 분노 공격 시 추가 스킬 발동 |
