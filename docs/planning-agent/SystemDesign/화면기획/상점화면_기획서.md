# 상점화면 기획서

## 화면 구성 (위 → 아래)

### 1-1. 스크롤 영역 (유연 높이 — 남은 공간 채움)
- 세로 스크롤 가능한 콘텐츠 영역
- 하단 탭에 따라 상품 카드 목록이 변경됨

### 1-2. 상점 탭 바 (고정 높이, 하단 고정)
```
[장비상점] [펫상점] [패키지상점] [충전]
```
- 4개 탭 전환
- 최초 진입 시 항상 "장비상점" 탭 활성

---

## 탭별 콘텐츠

### 장비상점

상품 카드 3종이 위에서 아래로 나열:

1. **보석 장비뽑기** (상단)
   - 일반~신화 전 등급 출현, S등급 장비 획득 가능
   - 보석으로 뽑기

2. **모험가 상자 뽑기** (중간)
   - 일반~우수 장비뽑기 상자
   - 은열쇠로 뽑기 가능

3. **영웅상자 뽑기** (하단)
   - 우수~에픽 장비 포함
   - 금열쇠로 뽑기 가능

### 펫상점

상품 카드 2종:

1. **우수 펫뽑기**
   - 에픽 이상 펫 뽑기 확률이 높은 뽑기
   - 보석으로 뽑기 가능

2. **일반 펫 뽑기**
   - 일반부터 드물게 에픽펫도 나오는 뽑기
   - 펫 알로 뽑기 가능

### 패키지상점
- 준비 중 (플레이스홀더)

### 충전
- 준비 중 (플레이스홀더)

---

## 상품 카드 구조

각 상품 카드의 공통 레이아웃:
- 상단 악센트 바 (등급 색상)
- 상품명 (볼드, 등급 색상)
- 설명 (일반, 약한 색상)
- 필요 재화 표시
- 1회/10회 뽑기 버튼 행

---

## 상호작용 흐름

1. 하단 탭 선택 → 해당 탭의 상품 카드 목록으로 스크롤 영역 갱신
2. 뽑기 버튼 클릭 → 재화 차감 + 결과 생성 + 인벤토리 추가 + 저장
3. 결과 팝업 표시 (GachaRewardPopup)
4. 팝업에서 "닫기" 또는 "한번 더" 선택

---

## 뽑기 결과 팝업 (GachaRewardPopup)

기존 구현 유지:
- 상단: 제목
- 중앙: 결과 그리드 (5열, 스크롤 가능)
- 하단: "닫기" / "한번 더" 버튼

---

## 데이터 매핑

```
보유 보석       → Player.resources.Gems
상자 시스템     → Game.EquipmentChestSystem
1회 비용        → chest.GetCostPerPull()
10회 비용       → chest.GetPull10Cost()
1회 뽑기        → Game.PullGacha()
10회 뽑기       → Game.PullGacha10()
```
